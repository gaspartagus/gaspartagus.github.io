trie.js

A cute little module for creating and searching trie hashmaps. It is designed for searching for sequences of words in a text.
For example the text "Hello, I'm John Coltrane" will be parsed as ["Hello"] and ["I'm", "John", Coltrane"] and hopefully, using a trie of artists, "John Coltrane" will be returned.

color.js

A bling-bling effect for html video elements.

In canvas the current frame of the video element can be passed to canvas.getContext("2d").drawImage(video) and the pixels can then be manipulated with context.getImageData(). This piece of code computes the mean color of the down-sampled current video frame, changes its saturation and luminosity and displays it in the background of the page.

Find it wrapped as Chrome extension for Youtube here : chrome-webstore-bling-your-tube :)

ludovic muncherji’s r7remote

This project uses the Arduino Yun board to try and remotely command the R7 webapp via websockets. The board runs a NFC module, and two independent processors, a basic - but IO capable - ARM processor, and a bigger one running a wifi linux distribution, OpenWRT.

With two processors come two scripts :

arduino.ino

This part deals with the IO part, the buttons, the lights and the NFC module. See the full graphcet : graphcet.png

Though not an international standard, the graphcet representation of an automaton is very useful for this type of application, as it allows to easily change the order of the actions, the timings and the logic behind it.

linino.js

On boot, the ARM part launches a node process in the linino using the arduino bridge library. This process listens for Chromecasts - or similar devices - on the network and attempts to connect to it when the users requires it. It then maintains a socket connection with the R7 webapp.

echo.sh

This shell script is an attempt to programmatically configure the wifi network the linino is connected to, as doing this manually is tedious and difficult. It works but sometimes requires to reboot the linux kernel.

The current version of the remote should be able to turn on and off the R7 app, as well as zap or switch it on with an NFC tag.

indexeddb api

IndexedDB is a powerful API for storing any javascript object along with as much indexed values as the users decides. It’s officially not persistent storage but in practice it is only wiped when the storage quota is reached.

In Chrome the quota is the whole disk space, divided by the number of databases declared in the browser. The Quota management API doesn’t apply to this type of storage as its officially temporary and temporary storage restriction can’t be modified by the user.

The js library aaronpowell’s db.js allows to create, query and update Indexed Databases very easily and in a cross browser fashion.

Basically anything can be stored in this type of storage, at least it worked for large ArrayBuffer blobs (2 MB).

filesystem api (chrome)

This API is only supported in Chrome but should not be deprecated as it is badly needed for the fastly growing Chrome apps. It allows to store only Blobs in a sandboxed filesystem, with no limit of size.

The storage quota needs to be requested programmatically although no user interaction is required.
